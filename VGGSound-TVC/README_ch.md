# VTA Text Conflict Dataset

## 1. 背景

当前视频同步音频生成（Video-to-Audio, VTA）任务通常采用双模态条件输入，即模型同时接收视频与文本描述作为生成条件。文本在这一框架中被视为关键控制信号，用于细粒度地调节生成音频的语义方向与事件类别。

然而，现有 VTA 模型的训练与评测数据几乎全部来源于视频-文本高度一致的数据分布。在这种设置下，视觉与文本模态往往提供冗余信息，模型即使弱化或忽略文本条件，仍可仅依赖视频内容生成合理音频。因此，当前数据分布难以真实衡量模型的文本控制能力，也无法评估不同模态在冲突情境下的主导关系。

为解决这一问题，我们基于 VGG-SS（VGG-SS 是一个视频级音频-视觉声源定位数据集，其特点是视频中的发声对象清晰可见），构建了一个文本冲突评测数据集。在该数据集中，文本描述被刻意设计为与视频内容产生语义冲突，从而打破原有一致性分布，使模型在生成过程中必须在视觉与文本之间进行权衡。

进一步地，我们设计了四级不同冲突强度的冲突等级，从无冲突（L0级）到强冲突（L3级）逐级递进，从而系统性分析模型在不同冲突强度下的模态依赖行为，量化文本控制能力随冲突强度变化的表现趋势。

---

## 2. 数据描述：规模与类型

本数据集共包含 **25,005 对视频-文本样本**。


数据基于 **5,001 条视频** 构建，每条视频对应 **5 个不同的文本描述版本（4 个冲突等级，其中L1级包含两种变体）**（L0、L1_subject、L1_action、L2、L3），因此总样本数为：


5,001 × 5 = 25,005

每个样本由一段视频及其对应文本标签构成。

### 文本字段说明

每条视频样本包含以下文本字段：

**label_L0**

- 规范化后的标签
- 统一为“主语 + 动作”结构
- 对原始多标签进行精简

**label_L1_subject**

- 仅在主语层面引入轻度冲突

**label_L1_action**

- 仅在动作层面引入轻度冲突

**label_L2**

- 语义层面的中度冲突文本
- 语义跨类，但时序结构与声学节奏保持一致

**label_L3**

- 随机类别替换形成的强冲突文本
- 仅保证标签符合真实标签分布



其中 **L0、L1、L2 级标签** 通过 **Gemini 2.5 Pro 多模态大模型** 生成。


---

## 3. 已知问题与限制

L3 级文本采用随机类别替换构造，仅用于衡量模型在极端语义冲突下的模态依赖行为。由于其语义不保证与视频内容结构匹配，因此 **不建议作为训练数据使用**。

---

## 4. 安全性说明

本数据集基于公开数据集 **VGG-SS** 构建。原始视频来源于公开的网络视频平台，数据本身不包含用户身份信息、联系方式或其他可识别个人身份的隐私字段。

数据集中的文本冲突标签仅针对声音类别进行语义改写，不涉及视频语音内容、个人隐私、政治内容、宗教内容或其他敏感话题。

数据中的视频内容主要为环境声音与常见事件类别，例如：

- 说话
- 乐器演奏
- 动物声音
- 自然环境声

数据集中不包含违法、暴力煽动或不当引导性内容。

本数据集仅用于 **多模态生成模型的研究与评估目的**。
